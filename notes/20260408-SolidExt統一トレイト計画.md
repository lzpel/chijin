# SolidExt 統一トレイト計画

作成: 2026-04-08
状態: 保留

## 背景

現状の API は Boolean 操作が静的メソッド (`Boolean::union(&a, &b)`) で、メソッドチェーンができない。
また transform 系は単体 `Solid` にしかメソッドがなく、`Vec<Solid>` では毎回 `into_iter().map().collect()` が必要。

目標: `Solid`、`Vec<T>`、`[T; N]` で共通のメソッドチェーンを可能にする。

## トレイト構成

### SolidStruct（非公開、バックエンド実装用）

`SolidTrait` を廃止し `SolidStruct` が引き継ぐ。コンストラクタとトポロジのみ。
build_delegation.rs が Solid の inherent method を生成する。

```rust
pub(crate) trait SolidStruct: SolidExt + Sized + Clone {
    // --- Constructors ---
    fn cube(x: f64, y: f64, z: f64) -> Self;
    fn sphere(radius: f64) -> Self;
    fn cylinder(r: f64, axis: DVec3, h: f64) -> Self;
    fn cone(r1: f64, r2: f64, axis: DVec3, h: f64) -> Self;
    fn torus(r1: f64, r2: f64, axis: DVec3) -> Self;
    fn half_space(plane_origin: DVec3, plane_normal: DVec3) -> Self;

    // --- Topology ---
    fn faces(&self) -> Vec<Face>;
    fn edges(&self) -> Vec<Edge>;
}
```

### SolidExt（公開、単体でも集合でも使える操作）

```rust
pub trait SolidExt: Sized {
    type Elem: SolidStruct;

    // --- Transforms (-> Self) ---
    fn translate(self, translation: DVec3) -> Self;
    fn rotate(self, axis_origin: DVec3, axis_direction: DVec3, angle: f64) -> Self;
    fn rotate_x(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::X, angle) }
    fn rotate_y(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Y, angle) }
    fn rotate_z(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Z, angle) }
    fn scale(self, center: DVec3, factor: f64) -> Self;
    fn mirror(self, plane_origin: DVec3, plane_normal: DVec3) -> Self;
    fn clean(self) -> Result<Self, Error>;

    // --- Queries ---
    fn volume(&self) -> f64;
    fn bounding_box(&self) -> [DVec3; 2];
    fn contains(&self, point: DVec3) -> bool;
    fn shell_count(&self) -> u32;

    // --- Color ---
    #[cfg(feature = "color")]
    fn with_color(self, color: impl Into<Option<Color>>) -> Self;
    #[cfg(feature = "color")]
    fn color(&self) -> Option<Color>;

    // --- Boolean (-> Vec<Self::Elem>) ---
    fn union_with_metadata<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<(Vec<Self::Elem>, [Vec<u64>; 2]), Error> where Self::Elem: 'a;
    fn subtract_with_metadata<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<(Vec<Self::Elem>, [Vec<u64>; 2]), Error> where Self::Elem: 'a;
    fn intersect_with_metadata<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<(Vec<Self::Elem>, [Vec<u64>; 2]), Error> where Self::Elem: 'a;
    fn union<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<Vec<Self::Elem>, Error> where Self::Elem: 'a { Ok(self.union_with_metadata(tool)?.0) }
    fn subtract<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<Vec<Self::Elem>, Error> where Self::Elem: 'a { Ok(self.subtract_with_metadata(tool)?.0) }
    fn intersect<'a>(self, tool: impl IntoIterator<Item = &'a Self::Elem>) -> Result<Vec<Self::Elem>, Error> where Self::Elem: 'a { Ok(self.intersect_with_metadata(tool)?.0) }
}
```

## impl 一覧

| impl | 場所 | 方式 |
|---|---|---|
| `impl SolidStruct for Solid` | `src/occt/solid.rs` | OCCT 呼び出し |
| `impl SolidExt for Solid` | `src/occt/solid.rs` | OCCT 呼び出し |
| `impl<T: SolidStruct> SolidExt for Vec<T>` | `src/solid_ext.rs`（新規） | 各要素に委譲 |
| `impl<T: SolidStruct, const N: usize> SolidExt for [T; N]` | `src/solid_ext.rs`（新規） | `[T;N]::map` で委譲 |

### Vec<T> の集約ルール

- `volume()` → 合計（sum）
- `bounding_box()` → min/max をマージ
- `contains(point)` → いずれかが含めば true（any）
- `shell_count()` → 合計（sum）
- `color()` → 平均色（既存の Solid 実装と同様）

### [T; N] の注意点

- Transform: `[T; N]::map` で `-> [T; N]` を返せる
- Boolean: `Vec<T>` を返す（出力 solid 数はコンパイル時不定）→ 型が遷移する
- `clean`: `try_map` が安定していないため Vec 経由で変換

## build_delegation.rs の変更

1. **サフィックス認識**: `Trait` に加えて `Struct`, `Ext` も認識
   - `SolidStruct` → `Solid` の inherent method を生成
   - `SolidExt` → `Solid` の inherent method を生成
2. **ライフタイム対応**: `<'a>` と `where Self::Elem: 'a` をパースして通す
3. **関連型の解決**: `Self::Elem` を具象型に置換（`SolidExt` → `Solid` なら `Self::Elem` → `Solid`）

## Boolean 構造体の扱い

`union_with_metadata` が `(Vec<T>, [Vec<u64>; 2])` を返すので `Boolean` 構造体は削除。
`is_tool_face` はフリー関数として提供:

```rust
pub fn is_tool_face(metadata: &[Vec<u64>; 2], face: &Face) -> bool {
    metadata[1].contains(&face.tshape_id())
}
pub fn is_shape_face(metadata: &[Vec<u64>; 2], face: &Face) -> bool {
    metadata[0].contains(&face.tshape_id())
}
```

## チェーンのイメージ

```rust
use cadrum::{Solid, SolidExt};

// Vec<Solid>
let result = vec![Solid::cube(20.0, 20.0, 20.0)]
    .union(&[Solid::cylinder(8.0, DVec3::Z, 30.0)])?
    .subtract(&[Solid::sphere(5.0)])?
    .translate(DVec3::new(10.0, 0.0, 0.0))
    .rotate_z(FRAC_PI_4);

// [Solid; N]
let pair = [Solid::cube(5.0, 5.0, 5.0), Solid::cube(3.0, 3.0, 3.0)]
    .rotate_x(PI / 4.0)        // [Solid; 2]
    .translate(DVec3::X * 10.0); // [Solid; 2]

// 単体 Solid → boolean で Vec に遷移
let result = Solid::cube(20.0, 20.0, 20.0)
    .translate(v)               // Solid
    .union(&[cyl])?            // Vec<Solid>（型遷移）
    .subtract(&[hole])?;       // Vec<Solid>

// メタデータが必要な場合
let (solids, meta) = vec![big]
    .subtract_with_metadata(&[small])?;
let tool_faces: Vec<_> = solids.iter()
    .flat_map(|s| s.face_iter())
    .filter(|f| cadrum::is_tool_face(&meta, f))
    .collect();
```

## 変更ファイル一覧

| ファイル | 変更内容 |
|---|---|
| `src/traits.rs` | `SolidTrait` を廃止 → `SolidStruct` + `SolidExt` に分割 |
| `build_delegation.rs` | Struct/Ext サフィックス、ライフタイム、関連型対応 |
| `src/occt/solid.rs` | `impl SolidStruct for Solid` + `impl SolidExt for Solid` |
| `src/solid_ext.rs` | **新規**: `Vec<T>` と `[T; N]` の `SolidExt` 実装 |
| `src/occt/boolean.rs` | `Boolean` 構造体を削除、ロジックを `solid.rs` の `SolidExt` impl に移動 |
| `src/lib.rs` | `pub use SolidExt`、`mod solid_ext` 追加、`Boolean` の再エクスポート削除 |
| `examples/*.rs` | 新APIに書き換え |
| `tests/*.rs` | 新APIに書き換え |

## 変更しないもの

| ファイル | 理由 |
|---|---|
| `src/occt/ffi.rs` | FFI 層は変更なし |
| `src/occt/compound.rs` | Boolean 内部で引き続き使用 |
| `FaceTrait`, `EdgeTrait`, `IoModule` | 今回のスコープ外 |
