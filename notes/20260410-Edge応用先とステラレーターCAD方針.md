# Edge 応用先とステラレーター CAD 方針

## 背景と最終目標

このプロジェクトの最終目標は **ステラレーター型核融合炉のプラズマ平衡面を CAD 化して OpenMC にそのまま投入し、中性子輸送シミュレーションが簡単に書けるようにすること**。

Edge API (helix, polygon, circle, 将来の bspline) の設計判断はすべてこの最終目標に結びつける。特に Edge は「閉じた曲線を sweep プロファイルにする」「ロフトの断面カーブ (rib) として並べる」という CAD ワークフローの最小単位なので、Edge コンストラクタの整備がステラレーター CAD の基礎になる。

## ステラレーター平衡面の数学的性質

- MHD 平衡計算 (VMEC, HINT 等) の出力はフーリエ級数表現:
  R(s, θ, φ) = Σ Rₘₙ(s) cos(mθ − n·Nfp·φ)
  Z(s, θ, φ) = Σ Zₘₙ(s) sin(mθ − n·Nfp·φ)
  (Nfp = field period 数)
- 非軸対称 (トカマクと違う): トロイダル方向に周期的に変形
- 内部的には解析的に滑らか
- トロイダルトポロジー (閉曲面、種数1)
- 入力は `wout_*.nc` (NetCDF) ファイル、もしくは CSV 経由

## 3つの実装経路と選定

| 案 | 方針 | B-spline 必要性 | 評価 |
|---|---|---|---|
| **A** | フーリエ係数から直接三角形メッシュを生成 → DAGMC 経由で OpenMC | 不要 | ❌ watertight manifold 保証が非自明、中間 CAD 編集不能 |
| **B** | 断面 rib 点群 → B-spline 閉曲線 → ロフトで閉曲面ソリッド | **B-spline 曲線のみ**。サーフェスは OCCT が自動構築 | ✓ **採用** |
| **C** | 点群 → B-spline サーフェス直接フィッティング → Face → Solid | **B-spline サーフェス必須** | ❌ ノット配置・周期性の扱いが難しい、OCCT のサーフェス API が重い |

### 選定理由: なぜ B 案か

1. **parastell (先行プロジェクト) が B 案を採用している** — 内部実装確認済み (後述)
2. **必要な新 API が最小** — `Edge::bspline(points, closed)` + `Solid::loft(sections, closed)` の 2 つで足りる
3. **OCCT が重労働を担う** — ロフト内部で B-spline サーフェスは自動構築。ノット配置・パラメタ化は OCCT デフォルトで十分な品質
4. **既存ライン (Edge → Wire → Face → Solid) に自然に載る** — sweep や boolean と同じレベルで扱える
5. **デバッグしやすい** — 中間段階の rib カーブを SVG で目視チェックできる
6. **拡張性** — 精度が足りなくなったら断面本数 N を増やすだけ

### parastell の実装確認 (B 案の証拠)

`parastell/build/lib/parastell/invessel_build.py` を確認した結果、完全に B 案。

**`Rib.generate_rib()`** (L1255-1263):
```python
rib_loci = [cq.Vector(tuple(r)) for r in self.rib_loci]
spline = cq.Edge.makeSpline(rib_loci).close()          # ← B-spline 曲線
rib_spline = cq.Wire.assembleEdges([spline]).close()    # ← wire 化
return rib_spline
```

**`Surface.generate_surface()`** (L1157-1164):
```python
self.surface = cq.Solid.makeLoft(
    [rib.generate_rib() for rib in self.Ribs]           # ← ロフト
)
```

処理の流れ:
1. 各トロイダル角 φᵢ でポロイダル点群 `rib_loci` を作る (フーリエ係数評価で)
2. `cq.Edge.makeSpline` で**閉じた B-spline 曲線 (Rib)** を作る
3. N 本の Rib を `cq.Solid.makeLoft` に渡して**ロフトで閉曲面ソリッド**を構築

**B-spline サーフェスを直接フィッティングする箇所は1つもない**。すべて OCCT の `BRepOffsetAPI_ThruSections` (ロフト) 内部で自動的にサーフェスが生成されている。CadQuery は薄いラッパーに過ぎず、OCCT の `Geom_BSplineCurve` + `BRepOffsetAPI_ThruSections` を呼んでいるだけ。

つまり parastell が動いている以上、**B 案は実証済み経路**。cadrum でも同じ 2 API を実装すれば同等のことができる。

## Edge コンストラクタのロードマップ

最終目標 (ステラレーター CAD) から逆算した Edge API の優先順位:

### 🔴 必須 (既実装 or 今すぐ)

| コンストラクタ | 状態 | 用途 |
|---|---|---|
| `helix` | ✓ 実装済み | コイルスパイン、ねじ山 |
| `polygon` | ✓ 実装済み | 閉じた直線断面 |
| `circle` | ✓ 実装済み | 円形穴、パイプ断面 |
| `line(a, b)` | ❌ 未実装 | 最も基本的な線分。構築ジオメトリ、直線断面 |
| `arc_3pts(start, mid, end)` | ❌ 未実装 | フィレット、丸角、部分円弧 |

### 🟠 ステラレーター作業の直前に実装

| コンストラクタ | 状態 | 用途 |
|---|---|---|
| `bspline(points, closed)` | ❌ 未実装 | **ステラレーターの rib カーブ (最終目標)** |

対で必要になる Solid 側 API:

| 関数 | 状態 | 用途 |
|---|---|---|
| `Solid::loft(sections, closed)` | ❌ 未実装 | **ステラレーター平衡面の構築 (最終目標)** |

### ⚪ 保留 (具体的需要が出てから)

- `ellipse` — 機械 CAD では稀
- `bezier` — bspline と重複、設計判断空間が広い
- `parabola` / `hyperbola` — ほぼ使わない
- `polyline` (開いた折れ線) — `line` があれば手動構築可

## 意思決定メモ

- **B-spline は最終目標に対して避けて通れない**。ただし「B-spline 曲線」までで十分で、「B-spline サーフェス」の直接 API は不要 (ロフト内部で自動構築される)。
- **parastell が B 案で成功している**ので、設計リスクはほぼゼロ。OCCT の該当 API (`GeomAPI_Interpolate`, `BRepOffsetAPI_ThruSections`) は枯れている。
- **ユーザー露出する API は最小限**。`Edge::bspline(points, closed)` と `Solid::loft(sections, closed)` の 2 つだけ。次数・ノット・パラメタ化などの設計判断はすべて OCCT デフォルトに任せる。必要ならあとから上位パラメータを追加できる。
- **AGENTS.md の「減らす方向優先」と整合**: trait method は 2 個増えるが、呼び出し側から見れば「3D フーリエデータを solid に変換する1本道」が提供されるだけで、ユーザー体験としては機能が増えたというより**ワークフローが短くなった**ように見える。
