# CAD 表現方式と中性子輸送計算の系統誤差

## 背景

ステラレーターの中性子輸送計算 (DAGMC + OpenMC など) では、プラズマ平衡面・ブランケット・遮蔽の CAD ジオメトリが入力になる。VMEC などから得られる平衡解は本来 $(\theta, \phi)$ 上の 2 次元グリッド点として与えられるので、そこから surface / solid を生成する CAD パイプラインに自由度がある。

このノートは、

1. **`GeomAPI_PointsToBSplineSurface::Interpolate` ベースのテンソル積補間**
2. **`BRepOffsetAPI_ThruSections` (loft) ベースの section 補間** (= ParaStell の現行方式)

の 2 方式を比較し、後者が中性子輸送結果に**CAD 起源の系統誤差**を持ち込むリスクを整理する。これは 20260410-loft閉じ方とparastell比較.md (closed loft の境界連続性) と 20260411-MakePipeShellのseam問題とGordonSurface案.md を補完する meta レベルの議論。

## 1. ParaStell の現行 CAD パイプラインの再確認

`parastell/invessel_build.py` を読むと、CAD 生成は次の 3 段で構成される。

### 段 1: section spline + loft

```python
spline = cq.Edge.makeSpline(rib_loci).close()
rib_spline = cq.Wire.assembleEdges([spline]).close()
self.surface = cq.Solid.makeLoft([rib.generate_rib() for rib in self.Ribs])
```

各ポロイダル断面を 1D B-spline 曲線にし、複数断面を CadQuery `makeLoft` (内部は OCCT `BRepOffsetAPI_ThruSections`) で繋ぐ。

### 段 2: 層構造のための boolean cut

```python
segment = outer_surface.cut(interior_surface)
```

外側 loft 面と内側 loft 面の差 (= FW / breeder / shield などの層) を取る。

### 段 3: 周期セグメントの rotate + fuse

```python
for angle in segment_angles:
    rot_segment = segment.rotate((0, 0, 0), (0, 0, 1), angle)
    component = component.fuse(rot_segment)
```

field period 1 つ分のセグメントを Z 軸回転させて全周フューズする。

## 2. ParaStell の boolean 使用の 2 分類

ParaStell が boolean を多用しているが、それぞれ性格が違う。

### (A) 層構造の cut: 構造的に必要

層 = 「2 面の間の体積」なので、`outer.cut(inner)` は CAD 表現方式に関係なく不可避。`Interpolate` 方式に置き換えても同じ cut が要る。これはアルゴリズムの限界ではなく問題設定そのもの。

### (B) 周期セグメントの rotate + fuse: ThruSections の限界の裏返し

field period $N$-fold 対称なら理屈の上では「$2\pi$ 全周分の点群を一発で `Interpolate` + `SetUPeriodic` する」だけで完結するはず。ParaStell が rotate + fuse しているのは、

- CadQuery の `makeLoft` が周期面を直接生成する経路を持っていない
- ThruSections に断面数 $\times N$ を渡そうとすると knot 互換化で制御点が爆発する
- 周期境界 ($\theta=0$ と $\theta=2\pi$) を「同じ断面」として渡すと OCCT が重複頂点エラーを出すことがある

という**実装上の制約への回避策**であって、本質的に必要な処理ではない。

20260410-loft閉じ方とparastell比較.md で論じた「OCCT は最初と最後の wire が `IsSame()` なら closed loft 経路に乗る」というトリックを使えば、cadrum 側では `Solid::loft(sections, closed=true)` 1 行で同じ閉曲面が作れる — つまり ParaStell の rotate + fuse は CadQuery ラッパー層の制約に縛られた回避策。

## 3. ThruSections 由来の波打ち

ParaStell の loft 方式は、たとえ closed loft トリックを使ったとしても、より深い問題を抱えている。

### v 方向の連続性が C¹ 止まり

`BRepOffsetAPI_ThruSections` (smoothed mode) は内部で `GeomFill_AppSurf` を使って全断面を一括 B-spline 近似する。u 方向 (各断面内) は次数 3 の C² が出るが、**v 方向 (断面間) の連続性は典型的に C¹ までしか保証されない**。

これは断面ごとの knot vector が独立に決まり、ThruSections が事後にそれを互換化するという構造に由来する。互換化のときに knot が「揃わない」場合、断面と断面の間にわずかな curvature 不連続が残る。

### Interpolate 方式との対比

`GeomAPI_PointsToBSplineSurface::Interpolate(pts)` は **2 次元グリッドの全点を一気にテンソル積補間**するので、u, v 両方向で C² が構造的に保証される。互換化のステップが存在しないので断面間で curvature が飛ぶことがない。

| 観点 | `Interpolate` | `ThruSections` |
|---|---|---|
| u 方向連続性 | C² | C² |
| v 方向連続性 | **C²** | **C¹** (典型) |
| 制御点数 | $(I+2) \times (J+2)$ | knot 互換化で肥大 |
| 周期境界 | `SetVPeriodic` で 1 コマンド | rotate + fuse 必要 |
| 入力点の通過 | 全点厳密 | 断面上の点は厳密、断面間は補間 |

### 波打ちの大きさ

定量的見積もりは難しいが、ステラレーター (minor radius $\sim 1\,\text{m}$) のスケールで **$0.1$ - 数 $\text{mm}$ オーダー**の表面凹凸が想定される。これは曲率の不連続点で局所的に強く出る。

## 4. 中性子輸送計算における verification gap

ここからが本題。**CAD 起源の波打ちが中性子輸送結果に効くかどうか**が問題。

### 波打ちが TBR に効く経路

ステラレーター中性子計算の主要量と波打ちの結合経路:

1. **TBR (Tritium Breeding Ratio)**: ブランケット経路長の局所変動 → 反応率の積分誤差。設計判断の基準は典型的に $0.5\%$ 程度の差で設計成立性が分かれる
2. **中性子壁負荷**: 表面法線のぶれ → 局所ピーク位置の量子化誤差
3. **遮蔽性能**: 層厚の局所変動 → 最薄部での leak 評価誤差
4. **熱発生分布**: 反応位置のずれ → 冷却設計の入力誤差

設計判断の有効ビット数 ($\sim 0.5\%$ TBR) と CAD 起源誤差 (波打ち $\sim 1\,\text{mm}$ → 経路長相対誤差 $\sim 10^{-3}$) は**同じオーダー**であり、原理的に「CAD ノイズで設計判断が flip する」可能性を排除できない。

### 検証ギャップとは何か

統計誤差 (Monte Carlo) は出力に出るので評価できる。メッシュ解像度依存性は収束テストで検証できる。しかし **CAD 表現由来の系統誤差は出力に出ない** — 同じ CAD で 2 回計算すれば同じ答えが返るので「再現性がある」と勘違いする。

これを検証するには「**別の CAD 表現方式で同じ点群から作った曲面で計算をやり直し、結果が一致するか見る**」しかない。ParaStell コミュニティでこの種のクロス検証が体系的に行われた形跡は、現時点では把握していない。

### 最悪の帰結

「設計 A vs 設計 B の比較で、物理的な差より CAD 数値ノイズの方が大きい」状況になると:

- 最適化ループは CAD ノイズの最小値を追ってしまい、物理最適から外れた解に収束する
- パラメトリックスタディの感度係数が CAD アーチファクト由来になる
- 査読論文の結論が「ThruSections のノット互換化アルゴリズムの実装詳細」に依存する

これは fusion 設計判断の根拠が**実装詳細にすり替わる**ことを意味し、当然望ましくない。

## 5. cadrum の意義

cadrum で `GeomAPI_PointsToBSplineSurface::Interpolate` ベースの曲面生成 API を実装する技術的意義は、単に「CAD 品質が良い」以上のものがある。

### 第 1: 独立な検証ツールチェーン

ParaStell の loft 方式と cadrum の Interpolate 方式は数学的に独立な経路なので、両者で同じ VMEC 入力から TBR を計算して比較すれば、CAD 起源系統誤差を直接定量化できる。**fusion neutronics の verification 文化に対する貢献**になり得る。

### 第 2: 周期構造の構造的保証

`SetVPeriodic` で field period の周期性を 1 コマンドで焼き込めるので、ParaStell の rotate + fuse 継ぎ目問題が構造的に消える。これは「ParaStell ではどう頑張っても得られない品質」の領域。

### 第 3: 層構造との両立

層構造の cut は anyway 必要だが、**「外側面と内側面をそれぞれ Interpolate で作って Solid 化してから cut」**するパスは、ParaStell の現行 cut と同じ呼び出しシーケンスで動く。つまり cadrum の Interpolate API を出しても下流への破壊的変更にはならない。

## 6. 検証ロードマップ (案)

cadrum で Interpolate API を実装したあとに行うべき検証実験:

1. **VMEC ベンチマーク**: 公開されている VMEC 平衡解 (W7-X, HSX, NCSX など) の点群を入力に、ParaStell loft / cadrum Interpolate の両方で平衡面を生成し、面の最大乖離 (Hausdorff 距離) を計測
2. **TBR クロス計算**: 同じ層構造設定で TBR を OpenMC で計算し、両方式の差を統計誤差と比較。差が統計誤差以下なら CAD 起源誤差は無視できる、それ以上なら verification gap が存在する証拠
3. **断面密度収束テスト**: ParaStell の rib 数を 2x, 4x に振って TBR の収束を見る。収束していなければ波打ちが効いている (Interpolate は構造的に収束済み)
4. **C² vs C¹ 比較**: 近似の `Interpolate` を v 方向 C¹ 設定でも作り、本来の C² 版との差を見ることで「連続性次数の違いだけが効いているか」を切り分け

これらは cadrum 単体では完結しないので、最低限 OpenMC との連携が要る。**fusion 物理 + CAD カーネルの両方を理解しているコミュニティは少数なので、cadrum がこの土俵に乗ることには独自性がある**。

## 7. 注意点

- **CAD 起源誤差が必ず大きい、と主張しているわけではない**。実測しないと分からない。verification gap という言葉は「測定されていない不確実性が存在する」という意味であって「破滅的な誤差がある」ではない
- **ParaStell の選択を批判しているわけでもない**。彼らのユースケース (CadQuery エコシステム + DAGMC ワークフロー) では loft が現実的な唯一解。問題は loft 自体ではなく**loft であることの不確実性が定量化されていない**こと
- **OCCT 8.0 への移行** (20260411-OCCT8移行計画.md) のタイミングでこの議論を実装に落とすのが自然。OCCT 8.0 は boolean まわりも改善されているので、層構造 cut の品質も同時に上がる可能性がある

## 8. このノートを書いた動機

ステラレーター CAD 用途として cadrum の Interpolate API を設計するときに、「ParaStell より良いものが作れる」という根拠を技術的にだけでなく**応用価値の観点から**整理しておきたかった。ThruSections の C¹ 限界が中性子輸送結果に伝播し得るという論点は、CAD カーネル開発の文脈だけでは出てこない (出力品質の議論止まり)。fusion neutronics の verification 文化と接続して初めて「なぜそれを直す価値があるか」が見える。

cadrum の存在意義の一つは「**OCCT を直接呼べる立場ゆえに、ラッパー層の制約に縛られない CAD パイプラインを fusion に提供すること**」にある。このノートはその主張の根拠を残しておくためのもの。
